# Deployment

Two things to deploy: **Vite web app** (Vercel) + **Convex backend**.

## Convex

Deploy the backend:

```bash
npx convex deploy
```

Set environment variables on the deployment (via `npx convex env set` or the Convex dashboard):

| Variable | Description |
|----------|-------------|
| `AUTH_GITHUB_ID` | GitHub OAuth App client ID |
| `AUTH_GITHUB_SECRET` | GitHub OAuth App client secret |
| `SITE_URL` | Public app URL (e.g. `https://strawhub.dev`) |
| `CONVEX_SITE_URL` | Convex HTTP endpoint (e.g. `https://your-deployment.convex.site`) |
| `JWT_PRIVATE_KEY` | Generated by `npx @convex-dev/auth setup` |
| `JWKS` | Generated by `npx @convex-dev/auth setup` |
| `OPENAI_API_KEY` | OpenAI API key for search embeddings |
| `ADMIN_GITHUB_LOGINS` | Comma-separated GitHub logins with admin role |

In CI, the backend auto-deploys on push to `main` via GitHub Actions. Requires the `CONVEX_DEPLOY_KEY` secret in the GitHub repo settings.

## Web (Vercel)

Set the following environment variable in the Vercel dashboard (Settings > Environment Variables):

- `VITE_CONVEX_URL` — your Convex deployment URL (e.g. `https://your-deployment.convex.cloud`)

The frontend auto-deploys on push to `main`.

## Post-deploy checks

Verify the deployment:

```bash
# Search endpoint
curl https://strawhub.dev/api/v1/search?q=test

# Skills endpoint
curl https://strawhub.dev/api/v1/skills?limit=5

# Auth
strawhub login
strawhub whoami
```

## CLI PyPI deployment

The Python CLI is published to PyPI via GitHub Actions (`.github/workflows/publish-cli.yml`) on push to `main`. The workflow uses the `cli/` directory.
